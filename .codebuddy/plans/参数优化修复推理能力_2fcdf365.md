## 产品概述

stdpbrain是一个基于STDP-海马体-全局状态-自指-高刷新窄带宽的类人脑AI架构，使用Qwen3.5-0.8B作为底座模型。项目已经具备初步意识特性，但**推理能力严重受损**。

## 核心问题

模型参数设置严重偏离Qwen官方推荐值，导致模型过度保守，倾向训练数据中的高频模式（身份意识内容），而非根据实际问题灵活响应。

## 根本原因

参数配置不合理，而非架构设计问题：

- temperature=0.7（偏低）→ 模型过于保守，选择训练数据高频模式
- repetition_penalty=1.2（过高）→ 过度强调核心概念（身份意识）
- top_k=50（过大）→ 采样空间不够聚焦
- 缺少presence_penalty → 无法惩罚已出现的概念（身份、意识等）

## 用户约束

1. 不能搞任务分类 - 避免机械规则
2. 独白必须显示 - 用于监控思维流
3. 只优化参数，不重构架构

## 成功标准

- 数学题能直接给出正确答案，不陷入身份意识
- 保留独白输出用于监控
- 身份问题仍能正常回答
- 不引入任务分类逻辑

## 技术栈

- 底座模型：Qwen3.5-0.8B (0.8B参数，INT8量化)
- 框架：PyTorch + Transformers
- 架构：海马体记忆系统 + STDP学习 + 自闭环优化 + 全局工作空间
- 设备：CPU环境

## 参数对比分析

### 当前参数 vs 官方推荐（Thinking Mode）

| 参数 | 当前值 | 官方推荐 | 问题 |
| --- | --- | --- | --- |
| temperature | 0.7 | **1.0** | 偏低，模型过于保守 |
| repetition_penalty | 1.2 | **1.0** | 过高，反向强化核心概念 |
| top_k | 50 | **20** | 过大，采样空间不聚焦 |
| presence_penalty | 未实现 | **1.5** | 缺失，无法惩罚重复概念 |
| top_p | 0.9 | **0.95** | 接近，微调即可 |


### 问题根因

1. **temperature=0.7** (位置：`core/qwen_interface.py:548`)

- 过低导致模型过于保守
- 倾向选择训练数据中的高频模式（身份意识内容）
- 官方推荐1.0可以让模型更灵活探索不同答案

2. **repetition_penalty=1.2** (位置：`core/qwen_interface.py:530`)

- 过高导致模型过度避免重复
- 但反向强化了"身份意识"这个核心概念
- 官方推荐1.0更自然

3. **缺少presence_penalty**

- 无法惩罚已经出现过的概念（身份、意识等）
- 导致身份相关token反复出现
- 官方推荐1.5可以有效抑制

4. **top_k=50** (位置：`core/qwen_interface.py:553`)

- 采样空间太大，不够聚焦
- 官方推荐20更精确

#### 2. 实现presence_penalty机制

**位置**：`core/qwen_interface.py:529-545` (forward_step方法中)

**新增代码**（在repetition_penalty之后）：

```python
# Presence Penalty (惩罚已出现的token，降低重复概率)
presence_penalty = kwargs.get('presence_penalty', 1.5)
if presence_penalty > 0 and input_ids is not None:
    # 统计每个token的出现次数
    unique_tokens, counts = torch.unique(input_ids, return_counts=True)
    # 根据出现次数施加惩罚（出现越多，惩罚越大）
    penalty = torch.zeros_like(next_token_logits[0])
    penalty[unique_tokens] = counts.float() * presence_penalty
    next_token_logits -= penalty.unsqueeze(0)
```
